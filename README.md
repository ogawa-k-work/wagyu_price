# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

* ローカルリポジトリ：自分のPC内にあるリポジトリ。自由に編集・コミットできる作業場所。

* リモートリポジトリ：GitHubなどのサーバー上にあるリポジトリ。チームで共有するための場所。

## プッシュとマージの違いは何でしょうか？

*	プッシュ：ローカルでの変更（コミット）をリモートリポジトリに送ること

*	マージ：あるブランチの変更を別のブランチに統合すること

## コミットとプッシュの違い

* コミット：ローカルリポジトリに変更履歴を記録すること

*	プッシュ：そのコミットをリモートリポジトリに反映させること

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

* 一目で変更内容がわかるようにする。

* 各チームやプロジェクトのルールに準拠したメッセージにする。

* なぜ変更したのかを書く。s

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

*	ローカルマージ：ローカルでコミットした後、自分のPC内でブランチを統合してその結果をプッシュする流れ。

*	プルリクエスト：ローカルでコミットした後、作業ブランチを一度リモートにプッシュしてチームメンバーのレビュー・承認を経てから統合する流れ。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？

*	どの変更が正しいかを確認する

*	コンフリクト箇所を手動で修正する

*	修正後に再度コミットする

*	チーム開発の場合は他メンバーと相談して解決する
